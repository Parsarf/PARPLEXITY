Implementation details (be exact)

1) Claim parsing (claim_parser.py)

Implement:

def split_into_claims(answer: str) -> list[str]
	•	If answer is bulleted/numbered, split by lines that start with:
	•	- , * , 1., 2., etc.
	•	Otherwise split into sentences using a simple heuristic:
	•	split on . , ? , !  with care for abbreviations (keep it simple)
	•	Strip whitespace and drop empty claims
	•	Return list of raw claim strings

def extract_citations(text: str) -> list[str]
	•	Parse citations of form [S1], [S2], possibly multiple: [S1][S3] or [S1, S3]
	•	Use regex to extract S\d+
	•	Return unique list in stable order

def remove_citation_markers(text: str) -> str
	•	Remove [S#] markers so verification uses clean claim text

2) Multi-source enforcement + citation coverage enforcement (citation_enforcer.py)

Goal: Ensure answer uses multiple sources and every claim has citations.

Implement:

def has_min_distinct_sources(claims: list[AnswerClaim], min_sources: int) -> bool
	•	Count distinct S# across all claims
	•	Return True if ≥ min_sources

def citation_coverage(claims: list[AnswerClaim]) -> float
	•	(# claims with ≥1 citation) / (total claims)

async def enforce_citations_and_multisource(...) -> str

This function may call OpenAI for a fix-up pass.

Inputs:
	•	query: str
	•	draft_answer: str
	•	context: str  # packed [S#] sources
	•	source_map: list[SourceRef]
	•	model, api_key
	•	thresholds:
	•	min_sources = 2 (default)
	•	min_coverage = 1.0 (require every claim/bullet cited)
	•	max_claims_single_source = 2 (avoid dominating one source)

Behavior:
	1.	Parse draft_answer into claims + citations.
	2.	If any claim has no citations OR distinct sources < min_sources OR one source dominates:
	•	Run one OpenAI “repair” call with very strict instructions:
	•	Keep the same content, do not add new facts not in sources.
	•	Add citations to every bullet/sentence.
	•	Ensure at least min_sources distinct sources used across the answer.
	•	Ensure no single source is cited in more than max_claims_single_source claims unless only one source exists.
	3.	Return repaired answer.
	4.	If already compliant, return original.

Note:
	•	If only 1 source exists in source_map, do not enforce multi-source rule.

3) Support verification (support_verifier.py)

This is a lightweight, deterministic verification that checks whether cited sources likely support the claim.

Implement:

def build_source_text_lookup(top_chunks: list[ScoredChunk], source_map: list[SourceRef]) -> dict[str, str]

Return mapping from S# → concatenated text from chunks belonging to that source id.
	•	Use the chunk→source_id mapping from your Phase 5 packer if available.
	•	If you do not have that mapping, approximate by matching chunk source_url to SourceRef.url.

def extract_keywords(claim_text: str) -> set[str]
	•	tokenize (lowercase alnum)
	•	drop stopwords + tokens < 3 chars
	•	return set of keywords (cap to 12–20)

def claim_supported_by_source(claim_text: str, source_text: str) -> tuple[bool, str]

Heuristic check:
	•	compute overlap between claim keywords and source keywords
	•	supported if overlap ≥ threshold (e.g., 2–3)
	•	additionally require at least one “anchor” term from claim appears in source_text (exact substring match)
Return:
	•	(supported True/False, support_notes)

def verify_claims(claims: list[AnswerClaim], source_lookup: dict[str, str]) -> list[AnswerClaim]

For each claim:
	•	if it has citations:
	•	mark supported=True if any cited source supports it
	•	otherwise supported=False and set support_notes
	•	if no citations:
	•	supported=False with notes “missing citation”

This verification must be deterministic and not call OpenAI.

4) Contradiction detection (contradiction.py)

Goal: If different sources imply different answers, report “disagreement”.

Lightweight approach:
	•	Only apply when query is factual/comparative and you have ≥2 sources.

Implement:

def detect_contradictions(claims: list[AnswerClaim], source_lookup: dict[str, str]) -> bool

Heuristic:
	•	Look for numeric claims or binary assertions (e.g., contains digits, “always/never”, “is/are”)
	•	If two claims about same subject appear with different numbers OR opposite polarity words, mark contradiction.
	•	Keep it simple: this is a flag, not a full contradiction engine.

Additionally implement:

async def add_disagreement_note_if_needed(...) -> str

If contradictions detected:
	•	Append a short “Sources disagree on …” section with citations
	•	Or request OpenAI to add a “Disagreements / nuances” section using only provided sources
Only one call maximum.

(If you choose no OpenAI call here, just add a short deterministic note and don’t fabricate specifics.)

5) Confidence scoring (confidence.py)

Implement:

def compute_confidence(claims: list[AnswerClaim], source_map: list[SourceRef], contradictions: bool) -> AnswerQuality

Rules:
	•	distinct_sources_used = count distinct citations across claims
	•	coverage = citation_coverage(...)
	•	unsupported_claims = count supported==False
	•	Confidence heuristic:
	•	HIGH if:
	•	distinct_sources_used ≥ 3
	•	coverage == 1.0
	•	unsupported_claims == 0
	•	contradictions == False
	•	MEDIUM if:
	•	distinct_sources_used ≥ 2
	•	coverage ≥ 0.8
	•	unsupported_claims ≤ 1
	•	LOW otherwise

Return AnswerQuality object.
